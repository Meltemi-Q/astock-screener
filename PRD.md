# A股「五层选股流水线」PRD

> 版本：v2.2 | 日期：2026-06-27 | 状态：已实现

## 一、产品概述

### 1.1 定位
面向个人价值投资者的 **一键式 A 股全市场自动化筛选工具**。把「排雷 → 质量 → 估值 → 安全边际 → 定性把关」五层选股法固化为脚本，对全市场约 5500 只 A 股自动打分排序，从 5000+ 缩到几十只候选。

### 1.2 核心原则
- **零依赖**：纯 Python 标准库，不装 akshare/tushare，不用 token
- **一键操作**：所有功能均可通过按钮完成，无需手动敲命令
- **数据可靠**：名称、价格、PE 等关键数据必须准确，不做脏数据回退
- **速度优先**：并行抓取 + 智能缓存，全市场 3-4 秒出结果

## 四、技术架构

### 4.1 文件结构
```
economy/
├── astock_screener.py    主筛选脚本（五层流水线 + HTML 输出）
├── stock_deep_dive.py    个股深度研报生成器
├── server.py             本地 HTTP 服务（使按钮可用）
├── layer4_report.py      第4层定性修正合并脚本
├── run.sh                一键入口（参数解析 + 编排）
├── README.md             使用文档
├── cache/                接口数据缓存（按 URL md5 存储）
└── results/
    ├── astock_screen_YYYYMMDD.html   交互式总表
    ├── astock_screen_YYYYMMDD.csv    全量数据
    ├── astock_shortlist_YYYYMMDD.md  Markdown 榜单
    └── deep_dives/
        ├── index.html               研报索引页
        ├── report.html              个股研报共享页面壳
        ├── assets/                  共享 CSS/JS
        └── data/
            └── XXXXXX.json          个股研报数据
```

### 4.2 数据流
```
东方财富公开接口
  ├── datacenter-web（财报/资产负债/商誉）──┐
  ├── push2delay（行情 PE/价格/市值）────────┤
  └── 腾讯财经（K线）───────────────────────┤
                                           ↓
                              ThreadPoolExecutor 并行抓取
                                           ↓
                              五层流水线判定 + 评分
                                           ↓
                          HTML / CSV / MD + JSON 研报数据输出
                                           ↓
                          server.py HTTP 服务 → 浏览器按钮
```

### 4.3 关键设计决策
| 决策 | 理由 |
|------|------|
| push2delay 批量抓取而非个股查询 | push2delay 的 `f12` 过滤参数不可靠，会返回其他股票数据。批量抓取+缓存是唯一可靠方式 |
| 名称从 CSV 传入而非 API 实时取 | push2 个股接口的 `f58` 字段返回脏数据，所有股票显示同一名称 |
| 并行度 6+5 | 金融数据源 6 个（财报+负债+商誉+3年历史）+ 行情 5 板块 |
| 缓存 6 小时 | 财报静态数据不常变，行情盘中会变但日频够用 |

## 五、已知限制

1. **AI 分析依赖 DeepSeek API Key**：需在 `~/.config/deepseek/api_key` 配置，无 key 时自动跳过
2. **K线数据来自腾讯财经**：腾讯接口不稳定时 K 线图为空
3. **金融股自然过滤**：银行/保险/券商被毛利≥30%+负债率<70% 自动排除，此选股框架不适用于金融股
4. **5 年财务数据**：当前取最近 5 年年报，如需更长可修改 `annual_inc[-5:]` 参数

## 六、数据口径

- 报告期：最近一期年报（非季报，避免 ROE/增速失真）
- PE：TTM（滚动 12 个月）
- 增长率：同比与 3 年 CAGR 取较小值（偏保守）
- 合理 PE = 增长率，限制在 [12, 30] 区间
- 合理市值 = TTM 净利 × 合理 PE

## 八、已知限制

1. **本地 file:// 限制**：`report.html` 通过 `fetch()` 读取 `data/XXXXXX.json`，本地直接双击打开可能被浏览器 CORS 拦截；推荐通过 `./run.sh` 的 HTTP 服务访问
2. **财年假设**：筛选器和研报均假设所有 A 股财年为 12 月 31 日，少数非标财年公司（如 3 月或 6 月年结）可能缺失最新年报数据
3. **CAGR 连续性**：3 年 CAGR 假设 4 个年报数据连续，不验证年份间隔
4. **嵌套线程池**：`fetch_stock_full` 内部调用 `fetch_spot_parallel()` 会再创建线程池，与外部 20 线程叠加可达 100 线程，实际对 I/O 等网络无影响但占内存
5. **服务器异常泄露**：500 错误直接返回 Python 异常消息到客户端（仅本地使用，低风险）
